# Sala de Juegos (TP #1)
## Enunciado
* Se debe realizar la aplicación “Sala de juegos”.
* La forma de corrección será por sprint de una semana.
* La aplicación debe permitir a nuestros usuarios (jugadores) medir sus capacidades cognitivas y motrices, debe ser simple de utilizar y permitir tener estadísticas de cada jugador y de cada juego.
* Los juegos deben guardar toda la información que sea importante para cada juego en particular.
* Solo se puede ingresar y jugar si es un usuario logueado o si se registra.
* Todos los juegos deben tener condiciones de finalización de victoria y de derrota claras.
* El TP cuenta con 4 sprints. El sprint #5 solo es obligatorio si el alumno debe entregar el TP en fecha de recuperatorios.

## Contenido de la aplicación
1. Aplicación frontend en Angular.
2. Servidor: Utilizar Supabase o Firebase.
3. Login y registro de usuarios: Autenticación y guardado de datos en base de datos.
4. Lógica de juegos:
    1. Ahorcado.
    2. Mayor o menor.
    3. Preguntados.
    4. Juego propio.
5. Juego propio: No deber ser ninguno de los siguientes:
    1. Tatetí.
    2. Memotest.
    3. Piedra, papel o tijeras.
6. Sala de chat:
    1. Debe ser una única sala que les permita a todos los usuarios registrados y logueados enviar mensajes y que se vean automáticamente (sin recargar la página).
7. Experiencia de usuario:
    1. Las pantallas deben contar con diseño trabajado y uniforme a lo largo de la aplicación.
    2. Navegación entre pantallas.
    3. Información clara y completa al mostrar mensajes o realizar acciones.
    4. Experiencia de usuario en los juegos. Tiempo de finalización / puntuación conseguida.
8. Diseño y estilos:
    1. Utilización de bootstrap, primeNG o cualquier librería / paquete de estilos.
    2. Utilización de animaciones (css y/o typescript).
    3. Favicon de la aplicación.
9. Listados de resultados.
10.Presentación. Página “Quién soy”:
    1. Datos personales del alumno.
    2. Imagen del alumno.
    3. Explicación del juego propio.
